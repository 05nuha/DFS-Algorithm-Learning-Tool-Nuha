DFS is both a search (explores depth-first) and a sort (outputs vertices in topological order by finish time). The animation below shows the sort step happening live.

🔍 Searching Role

DFS searches the graph for reachable vertices, diving as deep as possible before backtracking. It discovers all vertices reachable from each root and classifies every edge it encounters.

📋 Sorting Role

DFS produces a topological sort as a by-product: each vertex is prepended to the result list when it finishes. The result is sorted in decreasing finish-time order — valid for all DAGs.

Topological Sort — Watch the Sort Step Live

Each vertex is prepended to the result when it finishes (turns GREEN). Vertices with higher finish times appear earlier in the list.

⚠ What Goes Wrong Without Correct Sorting?

ScenarioResultVerdict
Remove topo sort — process in arbitrary orderDP relaxes before predecessors are final → wrong longest pathWRONG
Use BFS order instead of DFS finish orderVertices processed before all predecessors — DP invalidWRONG
Reverse the topo orderSuccessors before predecessors — all dist[] stay −∞WRONG
Sort by discovery time d[u]Differs from f[u] for forward/cross edges — partial errorsSOMETIMES WRONG
DFS finish-time sort (correct)Every predecessor processed first — DP always correctCORRECT ✓
Why f[u] is the right sort key: For any edge (u→v) in a DAG, DFS guarantees f[v] < f[u]. Sorting by decreasing f ensures u is processed before v for every edge — exactly what DP needs.
The Longest Path on a DAG uses Dynamic Programming — enabled by the topological sort that DFS produces. DP works here because the DAG structure eliminates cycles, guaranteeing subproblems can be solved in order.

✓ Dynamic Programming

Process in topo order. For each edge (u→v): dist[v] = max(dist[v], dist[u]+1). Each subproblem solved once. Predecessors always final before we process a vertex.

CORRECT — O(V+E)

✗ Greedy

Always pick the longest outgoing edge. Counterexample: S→A (w=10), A→T (w=1) vs S→B (w=1), B→T (w=100). Greedy picks S→A but misses S→B→T (total=101).

WRONG — misses global optimum

✗ Divide & Conquer

Split graph in half, solve each side, combine. The longest path may cross the cut boundary — subproblems are NOT independent. D&C gives wrong results.

WRONG — subproblems overlap

① Optimal Substructure

longest(s→v) = max over all u→v of (longest(s→u) + w(u,v)). The optimal solution is built from optimal sub-solutions.

② Overlapping Subproblems

dist[C] is updated from both A and B — watch for the purple* cells in the table above. Without memoization we'd recompute paths to C multiple times — exponential cost. DP stores each answer once and reuses it.

③ Topo Order Guarantee

Processing in topo order ensures dist[u] is already final when we process edge (u→v) — the core DP correctness property.
